Agent Simulation User Manual
1. Introduction

The Agent Simulation is an agent-based modeling program designed to simulate the "run and tumble" behavior of agents in an environment with food sources. Agents move, consume food, and adjust their behavior based on various parameters and environmental cues (like food frequency). The simulation offers both visual and headless modes, along with extensive configuration options and logging capabilities for analysis.

2. Getting Started

To launch the application, run the main.py file. This will open the graphical user interface (GUI), allowing you to configure and run simulations.

3. GUI Overview

The main window of the simulation GUI is organized into several tabs:

Controls & Sim Params: Configure global simulation settings, core physics, world setup, and agent-specific parameters.

Composite Config: (Enabled only when "Composite" agent type is selected) Configure the individual layers for the Composite agent.

Headless Runs: Set up and execute multiple simulation instances without a visual display for performance analysis.

Status & Graphs: View live simulation statistics, an event log, and performance history graphs.

4. Controls & Sim Params Tab

This tab allows you to configure the fundamental aspects of your simulation.

4.1. Global Modifiers & Timing

Target FPS: Sets the desired frames per second for the visual simulation, influencing the simulation's visual smoothness and the FRAME_DELTA_TIME_SECONDS. (Range: 10-240)

Global Speed Modifier: A scalar that globally adjusts the final calculated speed of agents. (Range: 0.1-5.0)

4.2. Core Motion Physics

Speed Scaling Factor: This constant controls how the total power generated by an agent translates into its forward speed. (Range: 1-100)

Angular Proportionality Constant: This constant scales the power difference between an agent's "sides" into a turn angle. (Range: 0.1-5.0)

4.3. World & Agent Setup

Import Preset... Button: Allows loading simulation configurations from a saved file (typically log files or specially prepared .txt presets).

Agent Type: Selects the behavioral model for the agents in the simulation.

Stochastic: Agents randomize their power outputs (P_r, P_l) at a specified interval.

Inverse Turn: Agents adjust their power outputs based on their current food consumption frequency, using sinusoidal waves.

Composite: Agents use multiple layers of sinusoidal wave generation, each activated based on a food frequency threshold.

# Agents: The number of simulated agents. (Range: 1-150)

Spawn X/Y: The initial X and Y coordinates where agents will appear on the screen. (Ranges: 0-WINDOW_W, 0-WINDOW_H)

Initial Orientation (°): The starting direction of agents in degrees. (Range: 0-359)

Food Preset: Choose from predefined food layouts:

three_lines_dense: Three horizontal lines of food.

void: No food in the environment.

filled_box_dense: A box-shaped area filled with food.

Draw Agent Trace: If checked, agents will leave a visual trail of their movement.

Permanent Trace: If checked along with "Draw Agent Trace", the agent's trace will not fade over time (its maxlen will be None).

Enable Logging: If checked, the simulation will record detailed agent movement and state data to a timestamped log file (.log) in the application directory.

4.4. Specific Agent Parameters (Dynamic)

This section changes based on the selected Agent Type:

For Stochastic Agents:

Re-randomize Interval (s): The time interval (in seconds) at which stochastic agents will generate new random power outputs for their left and right sides. (Range: 0.1-5.0)

For Inverse Turn Agents:

Turn Update Interval (s): The interval at which the agent makes a new turn decision. (Range: 0.1-5.0)

Thresholds:

Right Threshold (F/s): The food frequency (food items per second) below which the agent uses its "Low Food Conditions" for the right side, and above which it uses "High Food Conditions". (Range: 0.0-20.0)

Left Threshold (F/s): Similar to the Right Threshold, but for the left side. (Range: 0.0-20.0)

Right Side Controls / Left Side Controls: These sub-sections define the sinusoidal wave parameters based on food frequency.

Low Food Conditions: Applied when current food frequency is below the respective threshold.

Frequency (Hz): The frequency of the sinusoidal power wave. (Range: 0.1-10.0)

Amplitude: The amplitude of the sinusoidal power wave. (Range: 0.1-20.0)

High Food Conditions: Applied when current food frequency is above or equal to the respective threshold.

Frequency (Hz): The frequency of the sinusoidal power wave. (Range: 0.1-10.0)

Amplitude: The amplitude of the sinusoidal power wave. (Range: 0.1-20.0)

4.5. Action Buttons

Start Visual Sim: Begins the simulation with the current settings in a visual Pygame window.

Stop Visual Sim: Halts the currently running visual simulation.

Quit App: Exits the entire application.

5. Composite Config Tab

This tab is enabled only when "Composite" is selected as the Agent Type on the "Controls & Sim Params" tab. It allows you to define multiple "layer pairs" that contribute to the agent's total power output.

Add Layer Pair Button: Adds a new layer pair to the configuration.

Remove This Pair (X) Button: Removes the associated layer pair (at least one pair must remain).

Layer Pair Configuration: Each layer pair has independent settings for its left and right sides:

Threshold (F/s): The food frequency (food items per second) above which this specific layer contributes its sinusoidal wave to the total power. (Range: 0.1-30)

Amplitude: The amplitude of the sinusoidal power wave for this layer. (Range: -20 to 20)

Frequency (Hz): The frequency of the sinusoidal power wave for this layer. (Range: 0.1-10.0)

6. Headless Runs Tab

This tab facilitates running multiple simulation instances without a graphical display, useful for collecting data and comparing agent performance.

6.1. Headless Instance Configuration

Agent Type: Select the agent type for the headless runs.

Expire Timer (s): The duration (in seconds) for each individual headless simulation instance. (Range: 5-3600)

# Agents/Instance: The number of agents to simulate in each headless instance. (Range: 1-200)

# Instances: The total number of separate simulation runs to execute. (Range: 1-1000)

Start Headless Runs Button: Initiates the batch of headless simulations.

6.2. Results

Status: Displays the current state of the headless runs (e.g., "Idle", "Running instance X/Y").

Avg. Efficiency: Shows the average food items collected per second across all completed headless instances.

Median Efficiency: Shows the median food items collected per second across all completed headless instances.

7. Status & Graphs Tab
7.1. Live Sim Status

This section provides real-time information about the currently running visual simulation:

Simulation: Current status (Idle, Starting, Running).

Total Food: The total number of food items currently present in the environment.

Active Agents: The number of agents currently in the simulation.

Food Eaten: Cumulative food items consumed by all agents in the current run.

Efficiency (F/s): The average food items eaten per second since the start of the current run.

Input Freq (Hz): The current food consumption frequency of the primary agent, which drives the behavior of "Inverse Turn" and "Composite" agents.

7.2. Event Log

A text area that displays messages, warnings, and errors from the GUI and simulation, including log file status and headless run progress.

7.3. Performance History (Graphs)

(Requires matplotlib to be installed.) This area displays two real-time graphs during visual simulations:

Cumulative Metrics:

Plots Total Eaten (cumulative food collected), Decisions (cumulative turn decisions made by the primary agent), and Food Avail (food remaining in the environment) over time.

Performance Rate:

Plots the Consumption Rate (food items eaten per second, calculated over a sliding window) over time.

Analyze Log File... Button: Opens a dialog to select a .log file for post-simulation analysis. It extracts key statistics like total time, food eaten, distance traveled, efficiency, and path efficiency from the log.

8. Logging

When "Enable Logging" is checked in the "Controls & Sim Params" tab, the simulation creates a log file for each run.

File Name: raw_output_AGENTTYPE_FOODPRESET_TIMESTAMP.log

Content:

A header section with simulation configuration details (Agent Type, Food Preset, initial food count, and all relevant parameters). For Composite agents, the agent_params are logged as a JSON structure.

A data section with comma-separated values for each frame and each agent: frame,agent_id,pos_x,pos_y,angle_rad,delta_dist,delta_theta_rad.

A FINAL_STATS line at the end with the total food eaten.

9. Configuration Presets

You can save and load simulation configurations using the "Import Preset..." button. The log files generated by the simulation are themselves valid preset files, as they contain the full configuration in their header section.

10. Troubleshooting / FAQ

"Pygame Error": This typically indicates an issue with Pygame initialization or display. Ensure your graphics drivers are up to date and that your system meets Pygame requirements.

Graphs are not showing: Check if matplotlib is installed. If not, install it using pip install matplotlib.

Simulation freezes or crashes:

A critical error in the simulation thread might be occurring. Check the console for traceback messages.

Extremely high FPS settings combined with many agents or complex food layouts can strain system resources. Try reducing the FPS or number of agents.

Log file not created: Ensure the "Enable Logging" checkbox is checked before starting the simulation. Check for file permission issues in the application directory.

"Cannot Remove" warning for Composite Layers: The Composite agent requires at least one layer pair to function; you cannot remove the last one.
